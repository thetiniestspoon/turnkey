01 · Authentication
Two ways to get in

Turnkey has a private allowlist — only Shawn and Ted can log in. Two auth methods are available, and both establish a real Supabase session.

✉️

Magic Email Link

Enter your email, receive a one-click link, land on the dashboard. Standard Supabase OTP flow. Best for first login or session recovery.

🎯

Emoji PIN

Tap 4 emojis from a 25-emoji grid. Your unique sequence unlocks your account instantly — no email, no wait. Best for repeat use.

1

Tap 4 emojis

Frontend collects the sequence. 390,625 possible combinations.

2

SHA-256 hash sent to Edge Function

PIN is hashed client-side before leaving the browser. emoji-pin-auth function receives the hash.

3

Rate limit checked

3 failed attempts per IP = 5-min lockout. Prevents brute force.

4

User resolved, magic token generated

Hash matched against user_profiles.emoji_pin_hash. OTP token minted internally.

5

Session established

Client calls verifyOtp() — full Supabase session. Redirect to dashboard.

Default PINs — Shawn: 🦝🔑🏠💎  |  Ted: 🦊🎯🚀⭐    Both should change their PIN via Settings (gear icon) after first login.

03 · Scout
AI searches real listings for you

Enter a ZIP code, trigger the scout agent, and Claude searches Zillow, Redfin, and Realtor.com to find real properties that match your investment criteria.

1

Enter a ZIP code (or pick a saved territory)

Quick-select buttons show your watched markets. Type any ZIP to explore a new one.

2

agent-scout runs

Claude Sonnet 4.6 via LLM Gateway performs a web search against allowed real estate domains. Returns 3–8 real properties.

3

Results displayed as deal cards

Each card shows: address, price, beds/baths, score (0–100), recommended strategy (flip/rental), estimated ROI or cap rate, and listing URL.

4

Take action on a property

"Analyze with AI" → runs full financial analysis. "Add to Pipeline" → moves to Watching stage. "Watch" → saves to watchlist.

04 · Property Detail
The full deal — analysis, data, history

Every property has a deep-dive view showing financial projections, neighborhood context, market data, and the AI analysis history.

💰

Financial Analysis

Flip scenario: ARV, reno cost, carrying costs, profit, ROI, timeline.
Rental scenario: monthly cash flow, annual NOI, cap rate, cash-on-cash return. Both with confidence scores.

⚠️

Risk Badges

Visual flags for: flood zone status, market status (active/pending/sold), walk score tier. Know the risks before you analyze.

🏘️

Neighborhood Panel

Census demographics, median household income, unemployment rate (BLS), fair market rent (HUD), and comparable market context.

🚶

Walkability Gauge

Visual score from Walk Score API. High walkability = lower tenant vacancy for rentals.

🕐

Agent Timeline

Full history of every AI analysis run on this property — model used, tokens, cost, status, and output summary.

📈

Strategy Recommendation

AI recommends: flip, rental, or either. Color-coded badge. Recommendation stored with the analysis for future accuracy tracking.

08 · Predictions
Does the AI actually get it right?

Every AI analysis generates predictions (projected ARV, cap rate, ROI). This page tracks them against what actually happened — and shows the system's running accuracy.

🎯

Accuracy Score

Per-metric accuracy as a %. Color-coded: green (>80%), yellow (60–80%), red (<60%). The system gets smarter as you log actuals.

✏️

Inline Editing

Click any "actual" cell to enter the real outcome. Saving triggers agent-tracker to recalculate accuracy and update the pipeline record.

09 · Watchlists
Set it and let it scout for you

Create watched markets with investment criteria. Every night at 5 AM ET, agent-autoscout runs for each active watchlist and surfaces matching deals in your dashboard.

1

Create a watchlist

Pick a ZIP code, give it a name. Optionally override global criteria for this specific market.

2

Set global investment criteria

Max price, min cap rate (rental), min flip ROI, property types (SFR, MFR, condo, townhouse), minimum score (0–100), and strategies (flip / rental / either).

3

pg_cron runs at 5 AM ET nightly

Each watchlist's criteria merges global defaults + per-watchlist overrides. Staggered by day of week to avoid thundering herd on the API.

4

Matching deals land on your dashboard

New properties appear in "Recommended Deals" with a "New" badge (scouted within 24h). You watch, dismiss, or analyze.

10 · AI Agents
The engine under the hood

Nine Supabase Edge Functions run Claude via LLM Gateway. Each agent has one job. The orchestrator chains them. All runs are logged with model, cost, tokens, and status.

🔍 agent-scout
manual / cron — finds real listings by ZIP
📊 agent-analyst
manual — flip + rental financial projections
📡 agent-tracker
on actuals update — logs outcome vs prediction
🌎 agent-enricher
before analysis — Census, FRED, HUD, BLS, Walk Score
💬 agent-advisor
on demand — chat Q&A about deals and pipeline
🌙 agent-autoscout
5 AM ET nightly — proactive watchlist scouting
🔄 agent-orchestrator
coordinates agent dependency chains
📰 agent-market-check
periodic — flags stale listings (pending/sold)
📬 agent-digest
nightly — daily summary of new deals + changes
Model used throughout: Claude Sonnet 4.6 via ai-gateway.vercel.sh — confirmed live via health-check.
11 · Setup Guide
Getting Turnkey running from scratch

Everything you need to go from a fresh clone to a running system.

1 Clone and install

# Clone and install frontend deps
git clone https://github.com/thetiniestspoon/turnkey
cd turnkey
npm install

2 Create .env file

VITE_SUPABASE_URL=https://xebulbfhwyezjrqobzow.supabase.co
VITE_SUPABASE_ANON_KEY=[get from Supabase dashboard → Project Settings → API]

3 Set Supabase secrets (for Edge Functions)

Via Supabase Dashboard → Project → Edge Functions → Secrets, or via CLI:

supabase secrets set BLS_API_KEY=...
supabase secrets set CENSUS_API_KEY=...
supabase secrets set FRED_API_KEY=...
supabase secrets set HUD_API_TOKEN=...
supabase secrets set WALKSCORE_API_KEY=... # optional
supabase secrets set LLM_GATEWAY_URL=https://ai-gateway.vercel.sh/v1/chat/completions
supabase secrets set LLM_GATEWAY_API_KEY=...

See API-KEY-SETUP.md in the repo for where to get each key (most are free tier).

4 Run database migrations

supabase db push # runs all migrations in /supabase/migrations/

Five migrations: initial schema, watchlists/criteria, autonomous pipeline, pg_cron schedules, emoji PIN.

5 Deploy Edge Functions

supabase functions deploy # deploys all functions

6 Seed user profiles (emoji PINs)

After Shawn and Ted each complete their first magic link login, seed their emoji PIN hashes into user_profiles. Or let them set their own PIN via Settings immediately after first login.

7 Run locally or deploy

npm run dev      # local dev at localhost:5173
npm run build    # production build to /dist
# Vercel deployment is already wired — push to main to deploy

12 · UX Wishlist
What would make this noticeably better

Surfaced during the end-to-end review. Prioritized by user impact.

High Medium Low / Nice to have
01

Onboarding flow for new users

First login drops you directly on the dashboard with no context. A simple 3-step onboarding (set emoji PIN → set investment criteria → add first watchlist) would orient new users and immediately make the system useful. Ted needs this.

High
02

Prompt to change default PIN on first login

Default PINs (🦝🔑🏠💎, 🦊🎯🚀⭐) are hardcoded and known. No prompt exists to change them. First-login should show a "Set your PIN" interstitial before reaching the dashboard.

High
03

Agent run completion toasts / notifications

You trigger a scout or analysis and... nothing. No feedback that it's running, no toast when it completes. The Agent Pulse widget exists but it's not prominent enough. A lightweight notification system (or a pulsing indicator in the NavBar while agents run) would dramatically reduce uncertainty.

High
04

Contact linking to properties

Contacts and properties are separate data stores with no connection. You can't say "Mark Chen is the agent for this deal." Add a contact_id reference to the pipeline table, and show the linked contact on the property detail page.

Medium
05

Pipeline age indicator on kanban cards

No way to see how long a deal has been sitting in a stage. A subtle "14d" badge on kanban cards would show which deals are stalling and need attention.

Medium
06

Property comparison tool

No way to put two or three properties side by side. An "Add to compare" action that opens a comparison table (price, ARV, cap rate, ROI, risk flags) would help decide between competing deals.

Medium
07

Score explanation tooltip

Properties show a 0–100 score but the UI never explains what factors drive it. A tooltip or expandable section on the score badge (location: 25, market: 20, financials: 35, risk: 20) would build trust in the AI output.

Medium
08

Bulk actions on Recommendations

When autoscout surfaces 10+ deals, dismissing them one-by-one is tedious. A "Dismiss all" or checkbox multi-select + bulk action would keep the recommendations list from becoming noise.

Medium
09

Mobile kanban board

Drag-and-drop kanban boards are notoriously hard on touch screens. On mobile, the board needs a tap-to-move UI (tap card → show stage picker) as a fallback to drag-and-drop.

Medium
10

Walk Score unavailable — graceful fallback needed

The health-check shows walkscore: false — the API key isn't configured. The UI should gracefully hide or placeholder the walkability gauge when data is unavailable, rather than showing a broken state.

Medium
11

Advisor panel discoverability

The floating "Ask Advisor" orb is easy to miss. Consider surfacing it more prominently on the property detail page where a user most needs to ask questions ("What's a good offer price for this?").

Low
12

Market data staleness indicator

Market data is cached with an expiration. If a property's Census data is 6 months old, the analysis could be misleading. Show a "data freshness" indicator on the neighborhood panel and allow one-click re-fetch.

Low
13

Email or push notification for high-scoring autoscout finds

The autoscout runs at 5 AM but there's no way to know it found something good until you log in. A simple email (or even the existing agent-digest) should fire when a deal scores above your threshold.

Low
14

Dark/light mode persistence

If theme preference is stored only in memory (not localStorage or a user preference), it resets on every page load. Persist it to localStorage on toggle.

Low
15

Anon key embedded in assist-snapshot

The health-check curl command in assist-snapshot.json contains a Supabase anon JWT. Low-risk (anon keys have limited permissions) but worth rotating and replacing with a Supabase dashboard reference link instead of an embedded token.

Low
